Unbounded LLM costs

Every alert investigation uses multiple LLM calls. No per-tenant limits meant unpredictable infrastructure spend.

No visibility

Couldn't tell which tenant was consuming how much. Billing was a guessing game.

No graceful degradation

When budget ran out, everything just... kept running. No alerting, no blocking.

02 // the solution

Quota Service: Check Before Every LLM Call

Per-tenant quotas

Monthly LLM cost limits per tenant. Configurable by plan tier.

Real-time tracking

Every LLM call logged with model, tokens, cost. Daily aggregation for dashboards.

Smart blocking

When quota exceeded: block new investigations, alert admins, auto-unblock when reset.

Global quota

Platform-wide safety net. Even if individual quotas are generous, global cap prevents runaway.

03 // performance

Checked on Every LLM Call. Can't Be Slow.

In-memory caching

Cache quota state. Check cache first, DB second. Reduces DB queries by 90%.

DB-level daily aggregation

Don't SUM on every check. Pre-aggregate daily totals.

Throttled alerts

Don't spam email on every blocked request. Throttle to one alert per threshold breach.

Cost quota scalability

Optimized the hot path for tenants with 1000+ daily investigations.
