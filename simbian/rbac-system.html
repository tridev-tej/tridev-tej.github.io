No granular permissions

Admin or nothing. Can't give a SOC analyst write access to investigations but read-only to settings.

SAML role mismatch

Enterprise IdPs assign roles. We had no way to map those to platform permissions.

No audit enforcement

Permission checks were inconsistent. Some endpoints checked, some didn't.

02 // the design

Decorator-Based Enforcement

@rbac_with_audit(resource="soc_agent", permission="write") def update_investigation(request): ... Request --> Middleware --> @rbac_with_audit --> View | |--> Check role permissions |--> Log audit event |--> Return 403 with marker if denied

JSON-based permissions

Each role stores permissions as a JSON map. Simple to query, easy to extend.

{"soc_agent": "RW", "integrations": "R"}

Parent-child inheritance

Resources have hierarchy. soc_agent is parent of response_action - if you can't access the parent, children are blocked too.

Combined RBAC + audit

Single decorator does both. Every permission check is automatically logged. No way to skip audit.

03 // custom roles

Create Any Role, Map Any Permission

Before

  • 5 hardcoded roles
  • Admin, SOC Analyst, IT Agent, Manager, Viewer
  • No per-resource control
  • Same roles for every tenant

After

  • Unlimited custom roles
  • Granular resource-level R/W permissions
  • Per-tenant role definitions
  • SAML mapping per role

Custom role creation UI

Visual permission matrix. Toggle read/write per resource. Admins build roles without engineering help.

SAML role mapping

Map IdP roles to custom Simbian roles. Per-tenant mapping for MSSP customers with multiple orgs.

Legacy migration

Migrated all existing users to the new role system. Zero downtime. Old roles mapped to equivalent new permissions.
