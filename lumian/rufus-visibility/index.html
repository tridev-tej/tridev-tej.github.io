Anti-Detection

Five layers between us and a captcha.

L1 · Pacing Gaussian sleep 2-7s between queries, μ=4s, σ=1.5s. Tuned for realtime UX, still gaussian — no robotic intervals.
L2 · Cap 8 / minute Rolling 60s window cap, persisted across runs. Sahil's tested limit at single IP.
L3 · Session keepalive 1h auto-refresh Hourly systemd timer navigates Amazon tab to a random page (homepage / your-store / buy-again) then scrapes cookies + CSRF. Generates real traffic so Amazon sees the session as active.
L4 · Jitter Pre-query delay Random 0.5-3s pause before each request. Breaks robotic timing patterns at the runner.
L5 · Block detect Hard stop Captcha keywords → exit immediately. No retries through blocks.

Shipped + Next

Live at rufus.lumian.ai. One number per ASIN: RVS.

✓ Auto query generation. Enter ASIN → scrape title from /dp/<ASIN> → GPT-4o-mini generates 8 plausible shopper queries.

✓ Realtime visibility check. SSE streams every product Rufus returns per query as it lands. Target ASIN highlighted in the grid.

✓ Two scores per ASIN: RVS + Answer Fit. RVS (0-100, deterministic) = Coverage + Quality. Answer Fit (0-100, LLM-judged) asks GPT-4o-mini "does this product belong in the answer set Rufus returned?" — handles multi-correct-answer queries (8 valid iPhone 17 variants) where pure rank scoring is unfair. High Fit + Low RVS = right category, losing position. Low Fit everywhere = category mismatch.

→ Next: Cloudflare Access (email-gate @lumian.ai), BigQuery sink for RVS-over-time, multi-account scaling.
